## AstrBot shell/python 工具故障排查（2026-05-01）

**问题**：`astrbot_execute_shell` 和 `astrbot_execute_python` 报 `handler parameter mismatch` 错误。

**根本原因**：`computer_use_runtime` 配置为 `"none"`，导致工具无法初始化。

**解决方案**：
1. 修改配置文件：`computer_use_runtime` 从 `"none"` 改为 `"local"`
2. 同时把 `computer_use_require_admin` 改为 `false`
3. 完全重启 AstrBot：`systemctl stop astrbot && sleep 3 && systemctl start astrbot`

**教训**：修改配置后必须完全重启（stop+start），仅 restart 可能不够。

## ⚠️ 代码开发工作流（必须遵守）

### 流程
1. **编写代码** → 写入 `code/scripts/xxx.html`
2. **更新列表** → 运行 `python3 code/scan_scripts.py`
3. **预览确认** → 在编辑器中点击脚本加载并运行
4. **用户确认** → 用户检查效果后确认
5. **放入项目** → 确认后才复制到项目目录
6. **清空脚本** → 从 scripts/ 中删除已入库的脚本

### 命名规范
- `功能名.html`（如 `game-ui.html`）
- 英文小写+连字符

### 脚本列表更新命令
```bash
cd /root/data/workspaces/default_FriendMessage_3565912658/code
python3 scan_scripts.py
```
